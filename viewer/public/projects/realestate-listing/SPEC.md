# 부동산 매물장 디지털화 - 프로젝트 스펙 (PHVI)

> 생성일: 2026-03-07
> 프레임워크: 문제정의 → 가설 → 검증 → 개선 (PHVI 사이클)

## 문제 정의 (Problem)

### P1: 매물장 수기 작성 불편
부동산에서 매매/상가/전월세 3종류의 매물장을 수기로 작성하고 있어 모든 작업이 비효율적.
- **세부**: 고객 방문 시 매물 검색 비효율 (5~10분 소요)
- **세부**: 수정/추가 시 장부 훼손 및 가독성 저하
- **세부**: 3종 매물장 별도 관리 (각 15~18개 항목)
- **세부**: 저중요도 작업에 과다한 시간 소모 → 핵심 업무 잠식

### 매물장 항목 정리

**매매 매물장**: 기본 고객 정보, 건물구분, 도로명 주소, 상세 주소, 방문 경로, 용도지역, 층수, 주차장여부, 엘레베이터여부, 토지면적, 건물면적, 매가

**상가 매물장**: 기본 고객 정보, 상가종류, 상호명, 도로명 주소, 상세 주소, 방문 경로, 층수, 주차장여부, 엘레베이터여부, 토지면적, 건물면적, 보증금, 권리금, 월세, VAT, 관리비

**전월세 매물장**: 기본 고객 정보, 전/월세구분, 도로명 주소, 상세 주소, 방문 경로, 층수, 주차장여부, 엘레베이터여부, 전입일자, 계약일자, 확정일자, 임대인 정보, 임차인 정보, 보증금, 전세금, 월세금, 관리비

---

## 가설 (Hypothesis)

### H1: 매물장 디지털화
- **문제**: P1 매물장 수기 작성 불편
- **핵심 전제**: 디지털 입력/검색이 수기보다 압도적으로 빠르다
- **기대 효과**:
  - H1-1: 빠른 검색 기능 (조건별 필터링)
  - H1-2: 깔끔한 수정/이력 관리

### H2: 전용 웹앱 개발
- **문제**: P1 + 기존 솔루션 연 200만원 비용 부담
- **핵심 전제**: AI 활용 자체 개발로 비용 절감하면서 IT에 익숙하지 않은 사용자도 쓸 수 있는 직관적 UI 구현 가능
- **기대 효과**:
  - H2-1: 자체 개발로 구독 비용 절감
  - H2-2: PC + 모바일 반응형 (현장 매물 조회 가능)
  - H2-3: AI OCR로 기존 수기 데이터 이관

---

## 검증 계획 (Validation)

### V1: 프로토타입 도입 테스트
- **가설**: H2 전용 웹앱 개발
- **방법**: 3종 매물장을 모두 포함한 프로토타입을 실제 업무에 도입
- **성공 지표**:
  - V1-1: 매물 검색 시간 5~10분 → 수초 이내로 단축
  - V1-2: 전체 매물장 작업 시간 파격적 감소
  - V1-3: 대표 중개사님의 체감 만족도 ("이건 편하다")

## 개선 방향 (Improvement)

### I1: 정량 데이터로 설득
- **검증**: V1 프로토타입 테스트 결과
- **방향**: 사용 전후 시간 절약 데이터를 정량적으로 보여주어 지속 사용 설득

### I2: 기능 확장성 어필
- **방향**: 디지털이기 때문에 추후 통계, 알림, 자동화 등 기능 추가 가능성을 채택 근거로 활용
