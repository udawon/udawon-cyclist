# 부동산 매물장 디지털화 - 기능 요구사항 (PRD)

> PHVI 사이클 기반: 가설 = FR, 검증 = Acceptance Criteria

---

## FR1: 매물 등록/수정/삭제

3종 매물장(매매/상가/전월세)의 CRUD 기능.

### FR1-1: 매매 매물 등록
- 입력 항목: 기본 고객 정보, 건물구분, 도로명 주소, 상세 주소, 방문 경로, 용도지역, 층수, 주차장여부, 엘레베이터여부, 토지면적, 건물면적, 매가
- **AC**: 모든 필수 항목 입력 후 저장 가능, 저장된 데이터가 목록에 즉시 반영

### FR1-2: 상가 매물 등록
- 입력 항목: 기본 고객 정보, 상가종류, 상호명, 도로명 주소, 상세 주소, 방문 경로, 층수, 주차장여부, 엘레베이터여부, 토지면적, 건물면적, 보증금, 권리금, 월세, VAT, 관리비
- **AC**: 모든 필수 항목 입력 후 저장 가능

### FR1-3: 전월세 매물 등록
- 입력 항목: 기본 고객 정보, 전/월세구분, 도로명 주소, 상세 주소, 방문 경로, 층수, 주차장여부, 엘레베이터여부, 전입일자, 계약일자, 확정일자, 임대인 정보, 임차인 정보, 보증금, 전세금, 월세금, 관리비
- **AC**: 모든 필수 항목 입력 후 저장 가능

### FR1-4: 매물 수정
- 기존 매물 데이터를 수정하고 저장
- **AC**: 수정 시 기존 데이터가 유지된 상태에서 변경 항목만 수정 가능

### FR1-5: 매물 삭제
- 매물 데이터 삭제 (확인 다이얼로그 포함)
- **AC**: 삭제 전 확인 요청, 삭제 후 목록에서 즉시 제거

## FR2: 매물 검색/필터링

고객 방문 시 빠르게 매물을 찾을 수 있는 검색 기능.

### FR2-1: 통합 검색
- 고객명, 주소, 상호명 등 키워드로 3종 매물장 통합 검색
- **AC**: 키워드 입력 후 수초 이내 검색 결과 표시 (기존 5~10분 → 수초)

### FR2-2: 조건별 필터링
- 매물 유형(매매/상가/전월세), 가격대, 면적, 층수 등 조건 필터
- **AC**: 복수 조건 동시 적용 가능, 필터 결과 즉시 반영

### FR2-3: 매물 목록 정렬
- 등록일, 가격, 면적 등 기준으로 정렬
- **AC**: 오름차순/내림차순 전환 가능

## FR3: 반응형 디자인 (PC + 모바일)

사무실과 현장 모두에서 사용 가능한 반응형 웹앱.

### FR3-1: PC 레이아웃
- 넓은 화면에서 매물 목록 + 상세 정보를 한 화면에 표시
- **AC**: 1024px 이상에서 사이드바 + 메인 영역 레이아웃

### FR3-2: 모바일 레이아웃
- 스마트폰에서 매물 검색/조회가 편리한 레이아웃
- **AC**: 768px 이하에서 단일 컬럼 레이아웃, 터치 친화적 버튼 크기

---

## FR4: 직관적 UI (IT 비전문가 대상)

나이든 중개사도 쉽게 사용할 수 있는 단순한 인터페이스.

### FR4-1: 수기 매물장 유사 입력 흐름
- 매물장 양식과 유사한 폼 레이아웃으로 학습 비용 최소화
- **AC**: 중개사가 별도 교육 없이 매물 등록 가능

### FR4-2: 큰 글씨/버튼
- 가독성 높은 폰트 크기와 충분한 버튼 크기
- **AC**: 기본 폰트 16px 이상, 주요 버튼 높이 44px 이상

---

## FR5: 데이터 이관 (선택)

기존 수기 매물장 데이터를 디지털로 이관.

### FR5-1: AI OCR 이관
- 수기 매물장 사진을 촬영하여 AI 텍스트 인식으로 자동 입력
- **AC**: 인식된 데이터를 확인/수정 후 저장하는 워크플로우 제공

### FR5-2: 수동 입력 이관
- OCR 인식율이 낮을 경우 수동으로 기존 데이터 입력
- **AC**: 벌크 입력 UI로 빠른 수동 이관 지원
