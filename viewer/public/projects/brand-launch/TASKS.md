# 신규 브랜드 런칭 업무 프레임워크 — 할일 목록 (TASKS)

## 실행 순서 로드맵

```
① 런칭 체크리스트 → ② 역할별 책임 정의 → ③ 팀원 레벨 정립 → ④ 주간 피드백 루프
```

---

## Phase 1: 런칭 체크리스트 작성 [1순위]

- [ ] 브랜드 런칭 전체 프로세스를 단계(Phase)별로 구분
  - [ ] 기획 단계 업무 목록 작성
  - [ ] 제작 단계 업무 목록 작성 (콘텐츠, 디자인, 운영 준비)
  - [ ] 검수 단계 업무 목록 작성
  - [ ] 런칭 당일 업무 목록 작성
  - [ ] 사후 관리 업무 목록 작성
- [ ] 각 업무 항목에 담당 파트 / 담당자 배정
- [ ] 각 업무 항목에 마감일(타겟 데이트) 설정
- [ ] 구글 스프레드시트 또는 잔디에 체크리스트 업로드
- [ ] 팀 리드가 팀 회의에서 체크리스트 전체 안내

### 검증 할일
- [ ] 첫 런칭 후 일정 준수율 집계 (마일스톤 달성 여부 확인)
- [ ] 체크리스트 항목 완료율 측정

---

## Phase 2: 역할별 책임 정의 문서화 [2순위]

- [ ] 체크리스트 각 업무 항목에 주도자(1명) 지정
- [ ] 체크리스트 각 업무 항목에 협력자 지정 (복수 가능)
- [ ] 배분 이유 한 줄 메모 작성 (팀원 이해를 위한 근거)
- [ ] 팀 회의에서 역할별 책임 내용 전체 설명
- [ ] 팀원 이해 확인 표시 수집

### 검증 할일
- [ ] 개인 업무 이행률 측정 (체크리스트 기반)
- [ ] 팀원 피로도 체크 (쏠림 발생 여부 확인)

---

## Phase 3: 팀원 레벨 정립 (정성화) [3순위]

- [ ] 파트별 핵심 스킬 체크리스트 작성
  - [ ] 마케팅 파트 스킬 목록 정의
  - [ ] 디자인 파트 스킬 목록 정의
  - [ ] 운영 파트 스킬 목록 정의
- [ ] 업무 복잡도 등급 정의
  - [ ] A등급: 독립적으로 수행 가능
  - [ ] B등급: 협력이 필요한 업무
  - [ ] C등급: 지도·지원이 필요한 업무
- [ ] 팀원별 레벨 기록 + 판단 근거 메모 작성 (팀 리드)
- [ ] 팀 회의에서 레벨 기준 설명 (이해를 목표로)

### 개선 할일
- [ ] 레벨 기준에 대한 팀원 피드백 수집
- [ ] 첫 런칭 후 레벨 기준 재검토 및 보완

---

## Phase 4: 주간 피드백 루프 운영 [상시]

- [ ] 주간 회의 아젠다 템플릿 작성
  - [ ] 이번 주 진행 현황 공유 섹션
  - [ ] 지연/이슈 항목 논의 섹션
  - [ ] 재배분 결정 섹션
- [ ] 매주 팀 회의 시 체크리스트 현황 공유 정례화
- [ ] 재배분 내역 기록 (협업 툴)

### 개선 할일
- [ ] 피드백 흐름이 막힌 지점 파악 → 원인 분석
- [ ] 팀 리드 판단으로 업무 재배분 실행
- [ ] 분기별 프레임워크 전체 회고 및 개선
