# 업무 프레임워크 점검 - 기능 요구사항 (PRD)

> 생성일: 2026-03-06
> PHVI 사이클 기반: 가설 = FR, 검증 = Acceptance Criteria

---

## FR1: 상시 상주형 협업툴
데스크탑에 항상 떠 있는 위젯 형태의 협업툴. Sticky Notes처럼 가볍게 상주하며 업무 상태를 실시간 표시.

**수용 기준 (V1: MVP 직접 개발 필요)**
- [ ] 앱 실행 시 데스크탑 트레이에 상주
- [ ] 미니 위젯으로 현재 업무 상태 표시
- [ ] 클릭 시 전체 업무 목록으로 확장

### FR1-1: 작고 항상 보이는 UI
Windows Sticky Notes 크기의 미니 위젯. 별도 창 없이 항상 데스크탑에 존재.
- [ ] 카카오톡/스티키노트 사이즈의 기본 뷰
- [ ] 항상 최상위 표시 옵션
- [ ] 드래그로 위치 조정

---

## FR2: 상태변경 즉시알림
업무 상태 변경 시 사운드/비주얼 알림 자동 발생. PM이 별도 확인 없이 실시간 인지.

- [ ] 팀원 업무 상태 변경 시 PM에게 알림
- [ ] PM 피드백 시 팀원에게 알림
- [ ] 사운드 ON/OFF 설정

---

## FR3: 기록 동기부여 시스템
업무 기록 자체에 인센티브를 부여하여 자발적 상태 업데이트 유도.

**수용 기준 (V2: 2주 보상 실험)**
- [ ] 업데이트 횟수 지표: 실험 전 대비 증가 확인
- [ ] 피드백 소요시간 지표: 단축 확인

### FR3-1: 게이미피케이션 시스템
포인트/배지/랭킹을 협업툴에 내장하여 외부 보상 없이 자발적 동기 유지.

**수용 기준 (V5: 디지털→실물 보상 연결)**
- [ ] 디지털 보상만으로 동기 유지 여부 측정
- [ ] 실물 보상 연결 시 동기 변화 측정

#### FR3-1-1: 포인트/배지/랭킹
- [ ] 업무 완료 시 포인트 자동 적립 (PM 승인 후)
- [ ] 조건 달성 시 배지 자동 부여
- [ ] 팀원 간 기여도 랭킹 표시

#### FR3-1-2: 아바타 커스터마이징
- [ ] 기본 아바타 제공
- [ ] 파츠 상점 (머리/얼굴/상의/하의 등)
- [ ] 포인트로 파츠 구매 및 착용

#### FR3-1-3: 미니게임 포인트 소모
- [ ] 포인트로 미니게임 도전
- [ ] 낮은 확률로 실물 보상(커피권) 획득
- [ ] 포인트 소모 → 수집 동기 강화 루프

---

## FR4: 진입 비용 제로 설계
"들어가야 하는" 행위를 제거. 항상 업무 영역에 머물러 있어 도구 피로감 극복.

### FR4-1: 이벤트 기반 자연 접근
- [ ] 피드백 요청 시 위젯 내에서 즉시 확인+응답
- [ ] 포인트 구매 시 앱 전환 없이 처리
- [ ] 업무 할당 시 위젯에서 바로 수락/협의

### FR4-2: 사운드/비주얼 알림
- [ ] 중요 이벤트 시 사운드 알림
- [ ] 위젯 시각적 변화 (색상/애니메이션)
- [ ] 방해하지 않는 수준의 알림 강도 설정

---

## 비기능 요구사항

### PM 승인 시스템
- [ ] 완료 표기 → PM 승인 대기 → 승인 시 포인트 적립
- [ ] PM 미승인 시 사유 코멘트 필수

### 실물 교환 상점
- [ ] 포인트로 실물 보상(커피 교환권 등) 구매
- [ ] PM/리드가 상점 아이템 등록 및 관리

### 기술 스택
- **프레임워크**: Tauri (Rust 기반, 경량 데스크탑 앱)
- **개발 방식**: AI 협업 개발
- **타겟 OS**: Windows (우선)
