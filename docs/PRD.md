# Cyclist — 제품 요구사항 정의서 (PRD)

> **버전**: v1.0.0
> **최종수정일**: 2026-03-08
> **상태**: 진행 중 (MVP)

---

## 1. 제품 개요

### 1.1 제품명
**Cyclist** — Brainstorming Portfolio Viewer

### 1.2 한 줄 설명
PHVI 프레임워크로 정리한 브레인스토밍 결과를 마인드맵으로 시각화하고, 관련 문서를 한 화면에서 탐색하는 포트폴리오 뷰어

### 1.3 배경 및 문제 정의

| 문제 | 설명 |
|------|------|
| 브레인스토밍 결과 휘발 | 노션·피그잼·화이트보드 사진으로 흩어져 나중에 맥락 추적이 어려움 |
| 사고 흐름 단절 | 결론만 남고 "왜 그렇게 결론 냈는지" 과정이 소실됨 |
| 문서 산재 | SPEC·PRD·TASKS가 각각 다른 위치에 존재해 전환이 번거로움 |
| 공유 부재 | 구조화된 사고 과정을 이해관계자에게 보여줄 수단이 없음 |

### 1.4 해결 방법

React Flow 기반 **인터랙티브 마인드맵**으로 PHVI(문제정의→가설→검증→개선) 브레인스토밍 결과를 시각화한다.
노드 클릭 시 해당 생각의 출발점과 파생 흐름을 계보로 추적할 수 있고,
같은 화면에서 SPEC·PRD·TASKS 문서를 탭 전환으로 열람할 수 있다.

## 2. 목표 및 성공 지표

### 2.1 비즈니스 목표 (MVP)
- 브레인스토밍 결과를 마인드맵 + 문서 형태로 한 곳에서 탐색 가능하도록 구현
- 새 프로젝트 추가 시 JSON/마크다운 파일만 넣으면 자동 반영되는 구조 확보
- 포트폴리오 웹으로 배포하여 외부 공유 가능

### 2.2 성공 지표

| 지표 | 기준 |
|------|------|
| 마인드맵 초기 렌더링 | 2초 이내 (50노드 기준) |
| 프로젝트 추가 소요 시간 | 5분 이내 (JSON + md 파일 생성) |
| 마크다운 문서 로딩 | 1초 이내 |
| 초기 페이지 진입 | 3초 이내 (LCP) |

## 3. 대상 사용자

### 3.1 주요 사용자 그룹

| 구분 | 설명 |
|------|------|
| **프로젝트 소유자** | 브레인스토밍 결과를 정리하고 뷰어에 등록하는 사람 |
| **열람자** | 공유 링크를 통해 프로젝트 사고 과정을 탐색하는 사람 |

### 3.2 페르소나

**프로젝트 소유자 — "승원"** (PM, 여러 프로젝트 동시 진행)
- 프로젝트별 브레인스토밍 결과를 체계적으로 아카이빙하고 싶음
- 이해관계자에게 "왜 이 방향으로 갔는지" 사고 과정을 공유해야 함
- 복잡한 도구 설정 없이 파일만 넣으면 바로 반영되길 원함

**열람자 — "철수"** (팀원, 신규 합류)
- 프로젝트 배경과 의사결정 맥락을 빠르게 파악하고 싶음
- 마인드맵에서 전체 구조를 훑고, 관심 부분만 문서로 상세 확인
- 별도 로그인 없이 접근 가능해야 함

## 7. 데이터 모델 요약

```
public/projects/
├── index.json                 # 프로젝트 메타데이터 목록
└── {slug}/
    ├── mindmap.json           # 마인드맵 노드 배열
    ├── SPEC.md                # 프레임워크 스펙 문서
    ├── PRD.md                 # 제품 요구사항 문서
    └── TASKS.md               # 구현 태스크 목록
```

### 프로젝트 메타데이터 (index.json)

| 필드 | 타입 | 설명 |
|------|------|------|
| `slug` | string | URL 경로용 고유 식별자 |
| `name` | string | 프로젝트 표시명 |
| `description` | string | 프로젝트 한 줄 설명 |
| `createdAt` | string | 생성일 (ISO 8601) |
| `docs` | string[] | 제공 문서 목록 (SPEC, PRD, TASKS) |

### 마인드맵 노드 (mindmap.json)

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | 노드 고유 ID |
| `label` | string | 노드 제목 |
| `description` | string | 노드 상세 설명 |
| `category` | string | 카테고리 (문제정의·가설·검증·개선·사용자·경쟁사·위험요소·기술) |
| `parentId` | string \| null | 부모 노드 ID (루트는 null) |

```

    ├── /{slug}/spec       — SPEC 문서
    ├── /{slug}/prd        — PRD 문서
    └── /{slug}/tasks      — TASKS 문서

### 8.2 레이아웃 구조

```
┌─────────────────────────────────────────────┐
│  Header (로고, 테마 토글, 햄버거 메뉴)       │
├──────────────┬──────────────────────────────┤
│  Sidebar     │  <children />               │
│  ──────────  │                             │
│  프로젝트     │  [마인드맵 캔버스]            │
│   ├ brand    │   ├ 카테고리 탭 네비게이션    │
│   ├ sticky   │   ├ React Flow 캔버스       │
│   ├ task     │   └ 줌 컨트롤               │
│   └ estate   │                             │
│  ──────────  │  [상세 패널] (노드 선택 시)   │
│  문서 탭      │   ├ 노드 설명               │

│   ├ PRD      │                             │

## 11. 용어 정의

| 용어 | 설명 |
|------|------|
| **PHVI** | Problem-Hypothesis-Verification-Improvement. 4단계 사고 프레임워크 |
| **카테고리** | 노드의 성격을 구분하는 분류. PHVI 4종 + 보조 4종(사용자·경쟁사·위험요소·기술) |
| **계보** | 특정 노드의 조상(부모→루트)과 후손(자식→말단)을 연결한 전체 경로 |
| **Dagre** | 방향 그래프 레이아웃 엔진. 노드 좌표를 자동 계산 |
| **그룹 노드** | 같은 카테고리에 속한 노드들을 감싸는 시각적 컨테이너 |
| **상세 패널** | 노드 클릭 시 우측에 표시되는 설명·관계 정보 영역 |
| **slug** | URL 경로에 사용하는 프로젝트 고유 식별자 (예: `brand-launch`) |
| **Fit View** | 모든 노드가 화면에 들어오도록 줌·위치를 자동 조정하는 기능 |
