# 프로젝트/프로덕트 관리 프레임워크 - 개발 할일 목록

> 생성일: 2026-03-07
> 우선순위: A(파일럿) → B(1사이클) → C(장기) 순차 진행

---

## A. 파일럿 준비 (1개월)

### 스코어링 체계 설계
- [ ] 난이도 등급(1~5) 기준 정의
- [ ] 직군별 난이도 기준표 작성 (3개 직군)
- [ ] 완성도 평가 기준(1~5) 정의
- [ ] 과부하 임계치 기준 설정 (난이도×건수 합산)

### 스프레드시트 템플릿 구축
- [ ] 태스크 등록 시트 (태스크명, 담당자, 난이도, 마감일)
- [ ] 완성도 평가 시트 (완료일, 완성도 점수)
- [ ] 팀원별 대시보드 (누적 스코어, 부하량 자동 계산)
- [ ] 과부하 경고 조건부 서식 설정

### 팀 도입
- [ ] 팀원 대상 프레임워크 설명 및 도입 안내
- [ ] 1개월 파일럿 기간 설정 및 시작

## B. 1사이클 검증 (프로젝트 1바퀴)

### 데이터 수집 및 분석
- [ ] 파일럿 1개월 데이터 중간 점검
- [ ] 과부하 수치 감지 여부 확인
- [ ] 팀원 피드백 수집 (운영상 불편 사항)
- [ ] 프로젝트 1사이클 완료 후 전체 데이터 분석

### 임원 리포트
- [ ] 팀원별 기여도 리포트 작성
- [ ] 과부하 현황 리포트 작성
- [ ] 보상 요청 근거 자료 정리
- [ ] 임원 설득 전략 수립 (이탈 비용 vs 보상 비용 비교)

### 개선 사항 반영
- [ ] 난이도/완성도 기준 재조정 (파일럿 피드백 반영)
- [ ] 과부하 임계치 튜닝
- [ ] 배분 프로세스 개선
