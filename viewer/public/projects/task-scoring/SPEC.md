# 업무 분배 개선 방향성 - 프로젝트 스펙 (PHVI)

> 프레임워크: 문제정의 → 가설 → 검증 → 개선 (PHVI 사이클)

---

## 문제 정의 (Problem)

### P1: 우선순위 획일화 문제
같은 마감일이라도 개인 역량에 따라 체감 난이도가 달라, 일괄적인 우선순위 부여가 맞지 않는 구조적 문제.
- **세부**: 역량별 업무 부하 차이 → 우수 인력 과부하 딜레마 → 균등 배분 시 완성도 격차

### P2: 역량 파악의 비정량성
회의, 1:1 면담, 완성도 체크로 역량을 파악하지만 정량적/정성적 근거가 부족하여 배분 판단이 주관적.
- **세부**: 배분 근거 설명 곤란, 보상 체계 실행 어려움 (임원 설득 데이터 부재)

### P3: 실시간 피드백 부재
잔디와 스프레드시트를 사용 중이지만 업무 진행 상황에 대한 실시간 피드백이 어려워 상황 파악이 지연됨.

---

## 가설 (Hypothesis)

### H1: 업무 난이도 × 완성도 스코어링
- **문제**: P2 역량 파악의 비정량성
- **핵심 전제**: PM이 모든 실무를 파악하고 있다면, PM 단독 판단의 수치화만으로 충분한 근거가 됨
- **방식**: 태스크별 난이도(1~5)를 PM이 사전 부여, 완료 시 완성도 평가
- **기대 효과**:
  - H1-1: 임원 설득용 정량 데이터 생성
  - H1-2: 팀원에 대한 업무 배분 투명성 확보
  - H1-3: 직군별 난이도 기준표로 3개 직군 통합 관리

### H2: 커스텀 관리 툴 개발 (장기)
- **문제**: P3 실시간 피드백 부재
- **핵심 전제**: 스프레드시트 검증이 성공하면 전용 도구로 발전 가치가 있음
- **방식**: 업무 관리 + 스코어링 + 실시간 알림/피드백 통합 도구

---

## 검증 계획 (Validation)

### V1: 1개월 파일럿 테스트
- **가설**: H1 스코어링 프레임워크
- **방법**: 한 달간 실제 업무에 난이도/완성도 스코어링 적용
- **성공 지표**: 과부하 수치 감지 가능 여부 (특정 팀원의 난이도×건수 총합 임계치)

### V2: 프로젝트 1사이클 데이터 확보
- **가설**: H1-1 임원 설득용 데이터
- **방법**: 프로젝트 한 바퀴(기획~배포) 이상의 누적 데이터 수집
- **성공 지표**: 임원에게 제출 가능한 수준의 보고서 작성 가능 여부

---

## 개선 방향 (Improvement)

### I1: 데이터 기반 보상 요청
- **검증**: V1 과부하 수치 감지
- **방향**: 과부하 데이터를 근거로 해당 팀원의 승진/급여 보상을 임원에게 정량적으로 요청
- **기대**: 보상 → 동기부여 → 효율 상승 선순환 사이클

### I2: 반강제 도입 + 리더십 설득
- **상황**: 팀원 반발 시
- **방향**: 타협보다 리더십으로 설득. 과도한 타협이 팀 관리를 악화시킬 수 있음

### I3: 임원 설득 전략 수립
- **상황**: 임원 보상 거부 시
- **방향**: 데이터 제시 방식, 타이밍, 인력 이탈 비용 대비 보상 비용 비교 등 사전 준비

---

## 대상 사용자

- **팀 규모**: 8명 (PM 1명 포함)
- **직군**: 3개 직군
- **사용 범위**: 팀 내부 전용 도구 (범용성보다 실용성 우선)

---

## 리스크

| 리스크 | 심각도 | 대응 |
|--------|--------|------|
| 팀원 간 불만 발생 | 중 | 스코어링으로 배분 근거 투명화 |
| 핵심 인력 이탈 | 상 | 과부하 감지 + 보상 체계로 선제 대응 |
| 임원 보상 거부 | 상 | 이탈 비용 vs 보상 비용 비교 데이터로 설득 |

## 기술 환경

- **현재**: 잔디 (협업) + 스프레드시트 (업무 관리)
- **단기**: 스프레드시트에 스코어링 시트 추가
- **장기**: 커스텀 관리 툴 개발 (검증 성공 시)
