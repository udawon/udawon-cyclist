# Plan: 마인드맵 초기 뷰 개선 (mindmap-initial-view)

## 문제 정의

마인드맵에 처음 접속하면 전체 노드가 한 화면에 다 들어오도록 fitView가 적용되는데, 노드가 많을수록 (현재 sticky-office: 36개 노드, 8개 카테고리 그룹) 줌 레벨이 매우 낮아져서 글씨가 안 보일 정도로 축소된다. 사용자가 매번 줌인해야 내용을 읽을 수 있는 불편함이 있다.

### 현재 동작 분석

- `fitView` + `padding: 0.15`로 **모든 노드**를 뷰포트에 맞춤
- 그룹 레이아웃 전체 폭: Row1 그룹 6개 + Row2 그룹 4개가 수평으로 나열
- `minZoom: 0.3`이라 상당히 축소된 상태에서 시작 가능
- 결과: 첫 화면에서 노드 텍스트가 거의 읽히지 않음

## 개선 방향

### 옵션 A: 초기 줌 레벨 하한 설정 (추천)

`fitView`를 사용하되, `minZoom` 옵션으로 최소 줌 레벨을 제한한다. 전체가 안 보이더라도 읽을 수 있는 수준에서 시작하고, 사용자가 패닝으로 탐색한다.

```
fitViewOptions={{ padding: 0.15, minZoom: 0.55 }}
```

- 장점: 최소 변경, 텍스트 가독성 확보
- 단점: 노드가 적을 때 불필요한 여백 가능 (maxZoom으로 보완)

### 옵션 B: 첫 번째 그룹(문제정의)에 포커스

`fitView`의 `nodes` 옵션으로 Row1 첫 번째 그룹의 노드만 맞춘다. 사용자가 PHVI 흐름을 좌→우로 자연스럽게 탐색하도록 유도.

- 장점: PHVI 스토리라인의 시작점부터 볼 수 있음
- 단점: 나머지 노드의 존재를 처음에 인지하기 어려움

### 옵션 C: 미니맵 추가

React Flow의 `<MiniMap />` 컴포넌트를 추가해서 전체 구조를 축소판으로 보여주고, 메인 뷰는 적절한 줌 레벨로 시작.

- 장점: 전체 구조 인지 + 가독성 동시 해결
- 단점: UI 복잡도 증가

## 결정

**옵션 A + C 조합**을 추천한다.
- 초기 줌 레벨 하한으로 가독성 확보 (핵심)
- 미니맵으로 전체 구조 인지 보완 (부가)
