# 신규 브랜드 런칭 업무 프레임워크 — 기능 요구사항 (PRD)

## 제품 목표

신규 브랜드 런칭 시 마케팅·디자인·운영 3파트 8인 팀이 **체계적으로 업무를 분배하고 진행 현황을 관리**할 수 있는 실무 운영 프레임워크를 구축한다.

---

## FR1: 런칭 체크리스트

**[H1 가설 기반 / 1순위]**

### 설명
브랜드 런칭 전 단계별로 수행해야 할 업무를 체크리스트 형태로 사전 정의한다.

### 수용 기준 (Acceptance Criteria)
- [ ] 브랜드 런칭 전 과정을 단계(Phase)별로 구분하여 업무 목록이 작성되어 있다
- [ ] 각 업무 항목에 담당 파트/담당자가 배정되어 있다
- [ ] 구글 스프레드시트 또는 잔디에서 팀 전체가 열람 가능하다
- [ ] 팀 리드가 최초 배분 후 주간 회의에서 팀 전체에 안내한다
- [ ] 체크리스트 완료율을 기준으로 진행 현황을 파악할 수 있다

### 세부 기능
- **FR1-1**: 단계별 업무 분류 (기획 / 제작 / 검수 / 런칭 / 사후관리 등)
- **FR1-2**: 업무별 담당자 지정 칸
- **FR1-3**: 업무별 마감일(타겟 데이트) 입력 칸
- **FR1-4**: 완료 여부 체크 칸

---

## FR2: 역할별 책임 정의 문서

**[H2 가설 기반 / 2순위]**

### 설명
각 업무에 대해 "누가 주도하고, 누가 협력하는지"를 쉬운 언어로 명확히 정의한 문서를 작성한다. (RACI 개념을 쉽게 풀어낸 버전)

### 수용 기준 (Acceptance Criteria)
- [ ] 각 업무 항목에 주도자(1명)와 협력자(복수 가능)가 명시되어 있다
- [ ] RACI 같은 전문 용어 없이 팀원 누구나 이해할 수 있는 언어로 작성되어 있다
- [ ] 배분 이유가 최소 한 줄로 기록되어 있다
- [ ] 팀 회의에서 팀원 전원에게 설명이 이루어진다
- [ ] 개인 업무 이행률이 측정 가능하다

### 세부 기능
- **FR2-1**: 업무별 "주도자 / 협력자" 컬럼
- **FR2-2**: 배분 이유 메모 칸 (한 줄)
- **FR2-3**: 팀원 확인(이해) 표시 칸

---

## FR3: 팀원 레벨 기준표

**[H3 가설 기반 / 3순위]**

### 설명
업무 배분의 공정성을 확보하기 위해 팀원 역량을 정성적으로 기록하는 기준표를 작성한다.

### 수용 기준 (Acceptance Criteria)
- [ ] 파트별 핵심 스킬 체크리스트가 정의되어 있다
- [ ] 팀 리드의 판단 근거가 업무 복잡도 등급(A/B/C)으로 기록된다
- [ ] 각 팀원의 레벨에 따라 배정 가능한 업무 복잡도가 가이드된다
- [ ] 레벨 기준이 팀 회의에서 설명되어 팀원이 이해하고 있다

### 세부 기능
- **FR3-1**: 파트별 스킬 체크리스트 (마케팅 / 디자인 / 운영)
- **FR3-2**: 업무 복잡도 등급 정의 (A: 독립 수행 가능 / B: 협력 필요 / C: 지도 필요)
- **FR3-3**: 팀원별 레벨 기록 + 판단 근거 메모

---

## FR4: 주간 피드백 루프

**[V4, I1 기반 / 운영 프로세스]**

### 설명
매주 팀 회의에서 업무 진행 현황을 점검하고, 필요 시 팀 리드가 업무를 재배분하는 정기 루프를 운영한다.

### 수용 기준 (Acceptance Criteria)
- [ ] 주 1회 팀 회의에서 체크리스트 진행 현황이 공유된다
- [ ] 지연된 업무가 파악되고 원인이 논의된다
- [ ] 필요 시 팀 리드가 업무 재배분 결정을 내린다
- [ ] 회의 내용이 협업 툴(잔디/구글 시트)에 기록된다

### 세부 기능
- **FR4-1**: 주간 회의 아젠다 템플릿 (진행 현황 / 이슈 / 재배분 결정)
- **FR4-2**: 재배분 내역 기록 칸
