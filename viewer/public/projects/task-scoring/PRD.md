# 업무 분배 개선 방향성 - 기능 요구사항 (PRD)

> PHVI 사이클 기반: 가설 = FR, 검증 = Acceptance Criteria

---

## FR1: 업무 난이도 × 완성도 스코어링 시스템

업무 배분과 역량 파악을 정량화하는 핵심 프레임워크.

### FR1-1: 태스크 난이도 부여
- PM이 태스크 생성 시 난이도(1~5)를 부여
- 직군별 난이도 기준표 별도 관리
- **AC**: 모든 태스크에 난이도가 부여되고, 직군별 기준표가 존재할 것

### FR1-2: 완성도 평가
- 태스크 완료 시 PM이 완성도(1~5)를 평가
- **AC**: 완료된 태스크에 완성도 점수가 기록될 것

### FR1-3: 팀원별 기여도 대시보드
- 난이도 × 완성도 누적 스코어를 팀원별로 시각화
- **AC**: 팀원별 누적 스코어, 태스크 건수, 평균 난이도/완성도 확인 가능

### FR1-4: 과부하 감지 알림
- 특정 팀원의 난이도×건수 총합이 임계치를 초과하면 경고
- **AC**: 임계치 설정 가능, 초과 시 PM에게 알림 발생

---

## FR2: 임원 설득용 데이터 리포트

스코어링 데이터를 기반으로 임원 보고용 정량 리포트를 생성.

### FR2-1: 팀원 기여도 리포트
- 프로젝트 기간 동안의 팀원별 난이도×완성도 요약
- **AC**: 프로젝트 단위로 리포트 생성 가능, 팀원별 기여도 수치 포함

### FR2-2: 과부하 현황 리포트
- 특정 팀원의 업무 집중도와 과부하 이력을 데이터로 제시
- **AC**: 기간별 과부하 발생 빈도, 해당 팀원의 성과 데이터 포함

---

## FR3: 업무 배분 투명성

배분 근거를 팀원에게 설명할 수 있는 정량적 기반 제공.

### FR3-1: 배분 근거 조회
- 팀원이 자신에게 배정된 태스크의 난이도와 현재 총 부하량을 확인 가능
- **AC**: 팀원 본인의 현재 업무 부하(난이도 합산)를 조회할 수 있을 것

---

## FR4: 커스텀 관리 툴 (장기 목표)

스프레드시트 검증 성공 후 개발할 통합 도구.

### FR4-1: 실시간 알림/피드백
- 업무 상태 변경, 완성도 평가, 과부하 알림을 실시간으로 전달
- **AC**: 상태 변경 시 1분 이내 알림 수신

### FR4-2: 잔디 연동
- 기존 잔디 워크플로우와 연동하여 이중 관리 방지
- **AC**: 잔디에서 태스크 생성/완료가 스코어링 시스템에 자동 반영
